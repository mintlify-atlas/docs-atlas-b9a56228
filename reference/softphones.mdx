## What is a Softphone?

A softphone (combination of *software* and *telephone*) is software used for VoIP communications, either between softphones or to hardphones (physical phones for VoIP communications). They are very useful because you can test locally, without the need for additional hardware (which usually has a high price), the functionality of the PBX.

## Choosing a Softphone

### For Desktop Testing

- **Windows users:** Try **3CX** first for its stability and ease of use. Use **Blink** if you need TLS certificate support.
- **Cross-platform needs:** **Zoiper** is your best bet for testing across multiple operating systems.
- **Budget-conscious:** All listed desktop softphones have free versions suitable for Asterisk testing.

### For Mobile Testing

- **Best overall:** **Sipnetic** offers the most comfortable smartphone experience.
- **For messaging:** **MizuDroid** has better SIP message implementation.
- **Both are Android only** - iOS users should use Zoiper.

## Configuration Tips

<Note>
When configuring softphones to connect to your Asterisk PBX, you'll typically need:
- **Server/Domain:** Your Asterisk server IP address
- **Username:** The SIP extension (e.g., `javier`, `4000`)
- **Password:** The secret defined in `sip.conf`
- **Port:** Default is 5060 (UDP) or 5061 (TLS)
</Note>

### TLS Configuration

For secure connections using TLS certificates:

1. Generate client certificates as described in the [Call Security](/features/call-security) section
2. Use **Blink** on Windows for easiest certificate management
3. Configure the softphone to use TLS transport
4. Import the `.pem` certificate file

## Testing Scenarios

Softphones are ideal for testing:

- **Basic calling** between extensions
- **Voicemail** functionality
- **Call transfer** and forwarding
- **SIP messaging** (text messages)
- **Call pickup** and group calling
- **IVR menus** and dialplan logic
- **TLS/secure calling** (with Blink)

<Warning>
Softphones are excellent for testing but may not be suitable for production use in call centers or business environments where dedicated hardware provides better reliability and audio quality.
</Warning>
