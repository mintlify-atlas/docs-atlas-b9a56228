## Overview

This guide demonstrates a complete call center implementation for a health center (Centro de Salud), showcasing advanced Asterisk features including:

- Interactive Voice Response (IVR)
- Call queuing with multiple agents
- Database-driven user authentication
- Team-based call routing
- Voicemail integration
- Business hours scheduling

## Specifications

### General Requirements

- **Main phone number**: 757112233
- **Business hours**: Monday-Friday, 08:00-20:00
- **Outside hours**: Automated message with business hours
- **Hold music**: Custom music while waiting in queue

### User Registration Flow

1. Caller dials main number
2. IVR asks: Press 1 if registered, 2 if not registered
3. **Registered users**: Must enter 4-digit ID (verified against database)
4. **Unregistered users**: Route directly to reception
5. Music plays while waiting for available agent

### Team Structure

| Team | Extensions | Queue Strategy | Features |
|------|------------|----------------|----------|
| **Reception** | 101-103 (registered)<br/>110 (unregistered) | leastrecent | Route calls to other teams |
| **Direction** | 201-203 | leastrecent | Can capture calls from any team |
| **Doctors** | 301-303 | leastrecent | Team-only call capture |
| **Nurses** | 401-403 | leastrecent | Team-only call capture |

### Common Features (All Teams)

- TLS/SSL encryption for all connections
- Personal voicemail (access: `*<extension>`)
- Voicemail PIN = User ID number
- Call capture within team (except Direction)
- Group calling via `X00` extensions (e.g., 100 = all reception)
- External calling capability
- SIP messaging between extensions

## Configuration Files

### func_odbc.conf - Database Queries

This file defines ODBC functions to query the database:

```ini
[usuario]
dsn=asterisk
readsql=SELECT nombre from usuarios WHERE id='${SQL_ESC(${ARG1})}'

[extension]
dsn=asterisk
readsql=SELECT nombre from personal where extension='${SQL_ESC(${ARG1})}'

[pin]
dsn=asterisk
readsql=SELECT id from personal where extension='${SQL_ESC(${ARG1})}'

[equipo]
dsn=asterisk
readsql=SELECT lower(equipo) from personal where extension='${SQL_ESC(${ARG1})}'
```

### queues.conf - Call Queue Configuration

```ini
[general]
persistentmembers = yes
monitor-type = MixMonitor

[registrados]
strategy = leastrecent
timeout = 30
member => SIP/101
member => SIP/102
member => SIP/103

[noregistrados]
strategy = leastrecent
timeout = 30
member => SIP/110
```

**Key Parameters:**
- `strategy = leastrecent`: Calls the agent who hung up longest ago
- `timeout = 30`: Ring each agent for 30 seconds maximum
- `persistentmembers`: Queue members survive Asterisk restarts

## Key Features Explained

### 1. Business Hours Checking

```ini
exten => 757112233,1,GotoIfTime(8:00-20:00,mon-fri,*,*?abierto,1,1)
```

**GotoIfTime** syntax:
- `8:00-20:00`: Hour range
- `mon-fri`: Days of week
- `*`: Any day of month
- `*`: Any month
- `?abierto,1,1`: Jump to context `abierto`, extension `1`, priority `1` if true

### 2. Database Integration

The system uses ODBC to query the MariaDB database:

```ini
; Verify registered user
Set(nombre=${ODBC_usuario(${id})})

; Get extension owner name
Set(nombre=${ODBC_extension(${extension})})

; Get voicemail PIN
Set(idpin=${ODBC_pin(${llamante})})

; Get team for call pickup
Set(equipo=${ODBC_equipo(${extension})})
```

### 3. Queue Strategy

The `leastrecent` strategy ensures fair call distribution:
- Tracks when each agent last hung up a call
- Routes next call to agent with longest idle time
- Prevents call clustering on specific agents

### 4. Call Pickup Permissions

Different teams have different pickup capabilities:

| Team | Can Pickup |
|------|------------|
| Reception (1XX) | Only reception calls |
| Direction (2XX) | Any team's calls |
| Doctors (3XX) | Only doctor calls |
| Nurses (4XX) | Only nurse calls |

Implemented via prefix checking:
```ini
Set(pref=${extension:0:1})  ; Get first digit
GotoIf($["${pref}" = "3"]?llamar,8,1)  ; Allow only 3XX
```

## Monitoring and Troubleshooting

### View Queue Status

```
CLI> queue show registrados
```

Output shows:
- Current calls in queue
- Available agents
- Agent status (busy/idle)
- Call statistics

### View Active Calls

```
CLI> core show channels
```

### Debug Database Queries

```bash
sudo vim /etc/asterisk/logger.conf
```

Add:
```ini
[logfiles]
messages => notice,warning,error,verbose(3)
```

Then:
```
CLI> logger reload
CLI> core set verbose 5
```

## Performance Optimization

### Queue Tuning

Adjust based on call volume:

```ini
[registrados]
strategy = leastrecent
timeout = 30          ; Agent ring time
retry = 5             ; Seconds between retries
maxlen = 20           ; Maximum queue length
announce-frequency = 30  ; Position announcements
announce-holdtime = yes  ; Tell estimated wait
```

### Database Connection Pooling

```ini
[asterisk]
enabled => yes
dsn => asterisk-connector
username => asterisk
password => asterisk
pre-connect => yes
max_connections => 5  ; Connection pool size
```

## Scaling Considerations

- **Multiple queues**: Create separate queues per department for better isolation
- **Dynamic agents**: Allow agents to login/logout from queues
- **Call recording**: Enable MixMonitor for quality assurance
- **Real-time monitoring**: Integrate with queue monitoring dashboards
- **Overflow routing**: Route calls to voicemail after maximum wait time
