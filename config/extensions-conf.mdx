# extensions.conf Configuration

The `extensions.conf` file defines the Asterisk dialplan - the core call routing logic that determines how calls are processed, routed, and handled.

## File Location

```
/etc/asterisk/extensions.conf
```

## Configuration Structure

The dialplan consists of:
- **[general]** section - Global dialplan settings
- **[globals]** section - Global variables
- **Contexts** - Isolated call routing environments
- **Extensions** - Dial patterns within contexts
- **Priorities** - Sequential steps in call processing
- **Applications** - Actions to perform (Dial, Playback, etc.)

## [general] Section

```ini
[general]
static=yes
writeprotect=no
clearglobalvars=no
```

<ParamField path="static" type="boolean" default="yes">
  Use static configuration. Set to `no` to allow runtime dialplan modification
</ParamField>

<ParamField path="writeprotect" type="boolean" default="no">
  Prevent dialplan modification via CLI. Set to `yes` for production
</ParamField>

<ParamField path="clearglobalvars" type="boolean" default="no">
  Clear global variables on reload. Set to `yes` to reset on each reload
</ParamField>

## [globals] Section

Define global variables accessible throughout the dialplan:

```ini
[globals]
CONSOLE=Console/dsp
IAXINFO=guest
TRUNK=DAHDI/G2
TRUNKMSD=1
```

Access global variables with `${GLOBAL(variablename)}` or `${variablename}`.

## Dialplan Syntax

### Basic Structure

```ini
[context_name]
exten => extension,priority,Application(arguments)
```

### Components

<ParamField path="context" type="string" required>
  Named section that isolates call routing. Devices are assigned to contexts in their channel configuration (e.g., `sip.conf`)
</ParamField>

<ParamField path="exten" type="keyword" required>
  Keyword that defines an extension pattern or number
</ParamField>

<ParamField path="extension" type="string" required>
  The dialed number or pattern to match (e.g., `4000`, `_91NXXNXXXXXX`, `s`)
</ParamField>

<ParamField path="priority" type="integer" required>
  Execution order (1, 2, 3...) or `n` for next, or named labels like `n(label)`
</ParamField>

<ParamField path="Application" type="string" required>
  The action to perform (Dial, Answer, Playback, Hangup, etc.)
</ParamField>

## Extension Patterns

### Literal Extensions

```ini
exten => 4000,1,Dial(SIP/javier,15)
```

Matches exactly `4000`.

### Pattern Matching

Pattern matching uses underscore prefix `_` and wildcards:

```ini
exten => _4XXX,1,Answer()
exten => _91NXXNXXXXXX,1,Dial(${TRUNK}/${EXTEN})
exten => _X.,1,NoOp(Catch all)
```

<ParamField path="X" type="wildcard">
  Matches any digit 0-9
</ParamField>

<ParamField path="Z" type="wildcard">
  Matches any digit 1-9
</ParamField>

<ParamField path="N" type="wildcard">
  Matches any digit 2-9
</ParamField>

<ParamField path="[12345]" type="wildcard">
  Matches any digit in the brackets
</ParamField>

<ParamField path="." type="wildcard">
  Wildcard match for one or more characters
</ParamField>

<ParamField path="!" type="wildcard">
  Wildcard match for zero or more characters
</ParamField>

### Pattern Examples

| Pattern | Matches |
|---------|--------|
| `_4XXX` | 4000-4999 |
| `_NXXXXXX` | 2000000-9999999 |
| `_91NXXNXXXXXX` | 1 + area code + number (US) |
| `_*XXX` | *000-*999 |
| `_X.` | Any extension starting with a digit |
| `_[2-5]XX` | 200-599 |

## Priority Syntax

### Numeric Priorities

```ini
exten => 4000,1,NoOp(First step)
exten => 4000,2,Dial(SIP/javier,15)
exten => 4000,3,VoiceMail(4000)
exten => 4000,4,Hangup()
```

### Same (n) Priority

```ini
exten => 4000,1,NoOp(Calling Javier)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(4000)
same => n,Hangup()
```

The `same` keyword with `n` priority auto-increments.

### Named Labels

```ini
exten => 4000,1,NoOp(Starting)
same => n(dial),Dial(SIP/javier,15)
same => n,VoiceMail(4000)
same => n,Hangup()

; Jump to label
exten => 4001,1,Goto(4000,dial)
```

Use `n(labelname)` to create named priority labels for `Goto()` jumps.

## Common Applications

### Answer

Answer the call:
```ini
exten => 100,1,Answer()
```

### NoOp

No operation (useful for logging):
```ini
exten => 4000,1,NoOp(Calling Javier)
```

### Dial

Dial one or more devices:
```ini
exten => 4000,1,Dial(SIP/javier,15)
same => n,Hangup()
```

<ParamField path="technology/device" type="string" required>
  Channel to dial (e.g., `SIP/javier`, `DAHDI/1`)
</ParamField>

<ParamField path="timeout" type="integer">
  Ring timeout in seconds
</ParamField>

<ParamField path="options" type="string">
  Dial options (t=allow transfer, T=allow transfer to called party, r=ring tone, etc.)
</ParamField>

### Multiple Channels

```ini
exten => 4040,1,Dial(SIP/javier&SIP/belen,60)
```

Ring multiple devices simultaneously using `&`.

### VoiceMail

Send to voicemail:
```ini
exten => 4000,1,Dial(SIP/javier,15)
same => n,VoiceMail(4000)
same => n,Hangup()
```

<ParamField path="mailbox" type="string" required>
  Mailbox number or `mailbox@context`
</ParamField>

<ParamField path="options" type="string">
  - `u` - Unavailable message
  - `b` - Busy message
  - `s` - Skip instructions
</ParamField>

### VoiceMailMain

Access voicemail:
```ini
exten => *4000,1,VoiceMailMain(4000)
same => n,Hangup()
```

### Playback

Play sound file:
```ini
exten => 100,1,Playback(welcome)
```

Files are in `/var/lib/asterisk/sounds/` (without extension).

### Hangup

End the call:
```ini
exten => s,1,Hangup()
```

### Goto

Jump to another context/extension/priority:
```ini
exten => 100,1,Goto(empleado,4000,1)
```

### GotoIf

Conditional branching:
```ini
exten => _995554XXX,1,Set(extension=${EXTEN:5})
same => n,GotoIfTime(8:00-20:00,mon-fri,*,*?empleado,${extension},1)
same => n,Playback(closed)
same => n,Hangup()
```

### Set

Set channel variable:
```ini
exten => _4XXX,1,Set(extension=${EXTEN})
```

### Queue

Place caller in queue:
```ini
exten => 4444,1,Set(CHANNEL(musicclass)=recepcion)
same => n,Answer()
same => n,Queue(recepcion,,,,120)
same => n,Hangup()
```

## Context Types

### Simple Context

```ini
[empleado]
exten => 4000,1,NoOp(Calling Javier)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()

exten => 4001,1,NoOp(Calling Belen)
same => n,Dial(SIP/belen,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()
```

### Context with Include

```ini
[local]
ignorepa => 9
include => default
include => trunklocal
include => parkedcalls
```

The `include =>` directive includes all extensions from another context.

### Context with Switch

```ini
[dundi-e164-switch]
switch => DUNDi/e164
```

The `switch =>` directive enables dynamic lookups (e.g., database or DUNDi).

## Variables and Functions

### Channel Variables

```ini
exten => s,1,Set(extension=${EXTEN})
exten => s,2,Set(caller=${CALLERID(num)})
```

Common variables:
- `${EXTEN}` - Dialed extension
- `${CALLERID(num)}` - Caller's number
- `${CALLERID(name)}` - Caller's name
- `${CHANNEL}` - Current channel name
- `${DIALSTATUS}` - Result of last Dial (ANSWER, BUSY, NOANSWER, etc.)

### Variable Manipulation

```ini
exten => _995554XXX,1,Set(extension=${EXTEN:5})
```

- `${variable:offset}` - Substring from offset to end
- `${variable:offset:length}` - Substring with length

### CUT Function

```ini
exten => s,1,Set(remitente=${CUT(MESSAGE(from),<,2)})
same => n,Set(remitente=${CUT(remitente,@,1)})
```

Extract substring using delimiter.

### FILTER Function

```ini
exten => _9011.,1,Dial(${TRUNK}/${FILTER(0-9,${EXTEN:${TRUNKMSD}})})
```

Filter string to only allowed characters.

## Macro Definitions

Macros are reusable subroutines (deprecated in favor of GoSub in newer versions):

```ini
[macro-dundi-priv]
exten => s,1,NoOp(Macro dundi-priv de PBX180)
same => n,Goto(${ARG1},1)
include => dundi-priv-lookup
```

Call macro:
```ini
exten => _99999XXXX,1,Macro(dundi-priv,${EXTEN:5})
```

## Time-Based Routing

```ini
exten => _995554XXX,1,Set(extension=${EXTEN:5})
same => n,NoOp(Calling extension ${extension})
same => n,GotoIfTime(8:00-20:00,mon-fri,*,*?empleado,${extension},1)
same => n,Festival(Office is closed)
same => n,Hangup()
```

<ParamField path="GotoIfTime" type="application">
  Conditional goto based on time:
  - Format: `hours,days,days_of_month,months?context,extension,priority`
  - Example: `8:00-20:00,mon-fri,*,*` = 8am-8pm, Monday-Friday, any day of month, any month
</ParamField>

## Call Pickup

```ini
exten => _*8.,1,NoOp(Picking up call to extension ${EXTEN:2})
same => n,Pickup(${EXTEN:2})
same => n,Hangup()
```

Pick up ringing call to another extension.

## Message Handling

```ini
[mensajes]
exten => _X.,1,NoOp(Message from ${MESSAGE(from)})
same => n,NoOp(Message to ${MESSAGE(to)})
same => n,NoOp(Body: ${MESSAGE(body)})
same => n,Set(dest=${EXTEN})
same => n,Set(remitente=${CUT(MESSAGE(from),<,2)})
same => n,Set(remitente=${CUT(remitente,@,1)})
same => n,Set(remitente=${CUT(remitente,:,2)})
same => n,Set(texto=${MESSAGE(body)})
same => n,Set(MESSAGE(body)=${remitente}: ${texto})
same => n,MessageSend(sip:${dest}, Centro de Mensajes)
same => n,Hangup()
```

Handle SIP MESSAGE requests.

## Presence/Hints

```ini
[suscribir]
exten => 4000,hint,SIP/javier
exten => 4001,hint,SIP/belen
```

Hints enable presence/BLF (Busy Lamp Field) subscriptions.

## DUNDi Integration

```ini
[dundi-priv-canonical]
exten => _4XXX,1,Dial(Zap/g1/${EXTEN},20,rtT)

[dundi-priv-customers]

[dundi-priv-via-pstn]

[dundi-priv-local]
include => dundi-priv-canonical
include => dundi-priv-customers
include => dundi-priv-via-pstn

[dundi-priv-switch]
switch => DUNDi/priv

[dundi-priv-lookup]
include => dundi-priv-local
include => dundi-priv-switch

[macro-dundi-priv]
exten => s,1,Goto(${ARG1},1)
include => dundi-priv-lookup
```

## Complete Example

```ini
[general]
static=yes
writeprotect=no
clearglobalvars=no

[globals]
CONSOLE=Console/dsp
TRUNK=DAHDI/G2

[empleado]
; Employee extensions
exten => 4000,1,NoOp(Calling Javier)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()

exten => 4001,1,NoOp(Calling Belen)
same => n,Dial(SIP/belen,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()

; Voicemail access
exten => *4000,1,VoiceMailMain(4000)
same => n,Hangup()

exten => *4001,1,VoiceMailMain(4001)
same => n,Hangup()

; Call pickup
exten => _*8.,1,NoOp(Picking up call to ${EXTEN:2})
same => n,Pickup(${EXTEN:2})
same => n,Hangup()

; Queue
exten => 4444,1,NoOp(Reception queue)
same => n,Set(CHANNEL(musicclass)=recepcion)
same => n,Answer()
same => n,Queue(recepcion,,,,120)
same => n,Hangup()

[entrantes]
; Group call
exten => 4040,1,NoOp(Calling employee group)
same => n,Dial(SIP/javier&SIP/belen,60)
same => n,Hangup()

; Time-based routing
exten => _995554XXX,1,Set(extension=${EXTEN:5})
same => n,NoOp(Calling extension ${extension})
same => n,GotoIfTime(8:00-20:00,mon-fri,*,*?empleado,${extension},1)
same => n,Playback(office-closed)
same => n,Hangup()

[public]
include => empleado
```

## Testing Dialplan

Reload dialplan:
```bash
asterisk -rx "dialplan reload"
```

Show dialplan:
```bash
asterisk -rx "dialplan show empleado"
asterisk -rx "dialplan show 4000@empleado"
```

## Best Practices

1. **Use contexts** - Isolate internal and external call routing
2. **Use same => n** - More maintainable than numeric priorities
3. **Add NoOp for logging** - Helps with debugging
4. **Always Hangup** - Explicitly end calls
5. **Use variables** - Make dialplan more flexible
6. **Comment your code** - Use `;` for comments
7. **Test patterns** - Use `dialplan show` to verify pattern matching
